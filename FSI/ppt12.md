1. Transport Layer Security (TLS/SSL)
Originally evolved from the commercial SSL protocol, TLS is the primary method for securing internet transactions, such as e-commerce. It functions as a middleware between the Application layer and TCP.

2. Secure Shell (SSH)
SSH provides an authenticated and encrypted path to a remote operating system's command line over a network. It is the de facto replacement for insecure utilities like Telnet and rlogin.


3. Internet Protocol Security (IPSec)
Unlike TLS, which is application-specific, IPSec operates at the Network layer. This makes it transparent to applications, allowing all traffic crossing a perimeter (like a firewall) to be secured automatically
